# coding-1-finch-robot-project
Students will program a robot using microbit and MS MakeCode

Due date:

This midterm project challenges you to program a Finch robot to explore its environment autonomously using Python. Your robot must move on its own, detect obstacles with the distance sensor, respond by turning or stopping, and use LED colors to show its current state. You will provide your code and a short video to demonstrate your robot performance. You may work alone or in groups of up to three. When you are working with the finch robots, please leave the robots neatly packaged how you found them at the beginning of each class.

## Project Requirements:
## 1. Autonomous Movement (15 pts)
- Moves forward for a set period of time  
- Uses consistent speed controlled by a variable  
- Movement is smooth and intentional  

## 2. Obstacle Detection (15 pts)
- Correctly identifies when an object is too close  
- Responds by stopping, backing up, or turning  
- Resumes forward movement after avoidance maneuver  

## 3. Decision-Making Logic (10 pts)
- Code uses at least one loop  
- Code includes at least one conditional  
- Variables or structure make decisions easy to understand  

## 4. Visual Indicators (15 pts)
- Uses at least three distinct LED colors  
- Each LED color represents a meaningful robot state  
- LED changes at correct moments based on behavior  

## 5. Code Quality (10 pts)
- Code is organized into clear sections or functions where appropriate  
- Variable names are meaningful and descriptive  
- Includes at least 5 thoughtful comments explaining logic or intent  
- No unused or duplicate code left in the final version  
- Code runs without errors or warning messages  

## 6. Reflection (20 pts)
> Each group member must write their own reflection

- Paste a snippet of code from your project that you personally wrote  
- Explain how this code works and why you wrote it that way  
- Describe a bug or unexpected behavior you encountered  
  - What caused it in your code, and how did you fix it (or attempt to fix it)?  
- How does your robot decide when to turn or continue moving?  
  - Explain the exact sensor values and logic your code uses to make that decision  
- Reflection is at least **200 words**  

## 7. Video Documentation (15 pts)
- Record one video per group showing your robot running your code  
- The video is **2–3 minutes long**  
- Shows the robot moving independently without repositioning  
- Clearly demonstrates requirements outlined above  
- The narrator clearly explains robot movements  

---

## Optional Bonus (+10 pts each)
> Indicate bonus work with comments in your code

- Add randomized turns or speeds so the robot explores unpredictably  
- Print a status log each cycle showing distance, LED color, and decision making  
- Use five or more LED colors to represent different robot modes  
- Use the light sensor to detect “safe zones” and move toward or away from them  
- Program a “return to start” behavior after the program finishes  
- Add a celebration dance (movement + LEDs) when the program completes successfully
